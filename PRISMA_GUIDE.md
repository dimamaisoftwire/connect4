# Using Prisma with Connect 4

This project uses **Prisma ORM** for database modeling and queries. Prisma provides SQLAlchemy-like functionality for Node.js/TypeScript with excellent type safety and developer experience.

## What is Prisma?

Prisma is a next-generation ORM that provides:

- **Type-safe** database client (auto-generates TypeScript types)
- **Declarative data modeling** (similar to SQLAlchemy) in `schema.prisma`
- **Automatic migrations** for schema changes
- **Visual database explorer** (Prisma Studio)
- **Zero-runtime overhead** queries

## Project Structure

```
prisma/
├── schema.prisma          # Your database model definitions
└── migrations/            # Autogenerated migration files
app/
└── lib/
    └── prisma.ts          # Prisma Client singleton
```

## Database Model

The current schema defines a `Game` model:

```prisma
model Game {
  id        Int     @id @default(autoincrement())
  winner    Int     // 1 = Player 1, 2 = Player 2, 0 = Draw
  loser     Int     // 1 = Player 1, 2 = Player 2, 0 = Draw
  moveCount Int
  createdAt DateTime @default(now())
}
```

This maps to the `games` table in PostgreSQL.

## Working with the Database

### Using Prisma Client

```typescript
import { prisma } from "@/app/lib/prisma";

// Create a game
const game = await prisma.game.create({
  data: {
    winner: 1,
    loser: 2,
    moveCount: 42,
  },
});

// Read games
const games = await prisma.game.findMany({
  orderBy: { createdAt: "desc" },
  take: 10,
});

// Update a game
const updated = await prisma.game.update({
  where: { id: 1 },
  data: { winner: 2 },
});

// Delete a game
await prisma.game.delete({
  where: { id: 1 },
});
```

### Type Safety

Prisma automatically generates types from your schema:

```typescript
import { Game } from "@prisma/client";

const game: Game = await prisma.game.findUnique({
  where: { id: 1 },
});

console.log(game.winner); // TypeScript knows this is a number
```

## Common Tasks

### View Your Database

```bash
npm run db:studio
```

Opens Prisma Studio at `http://localhost:5555`

### Extend the Schema

Edit `prisma/schema.prisma`, then:

```bash
npm run db:migrate
```

Example - Add player names:

```prisma
model Game {
  id        Int     @id @default(autoincrement())
  winner    Int
  loser     Int
  moveCount Int
  createdAt DateTime @default(now())
  player1Name String?  // Optional field
  player2Name String?
}
```

Then run `npm run db:migrate` and choose a name like "add_player_names".

### Add Relations

Example - Connect games to a Player model:

```prisma
model Player {
  id    Int     @id @default(autoincrement())
  name  String  @unique
  games Game[]
}

model Game {
  id        Int     @id @default(autoincrement())
  winner    Player  @relation("Winner", fields: [winnerId], references: [id])
  winnerId  Int
  loser     Player  @relation("Loser", fields: [loserId], references: [id])
  loserId   Int
  moveCount Int
  createdAt DateTime @default(now())
}
```

### Query with Relations

```typescript
const gamesWithPlayers = await prisma.game.findMany({
  include: {
    winner: true,
    loser: true,
  },
});
```

## Best Practices

1. **Always check types** - Lean on TypeScript's type checking
2. **Use transactions** for related operations:

   ```typescript
   await prisma.$transaction([
     prisma.player.create({ data: { name: "Alice" } }),
     prisma.game.create({ data: { winnerId: 1, loserId: 2, moveCount: 30 } }),
   ]);
   ```

3. **Use `.select()` or `.include()`** to optimize queries:

   ```typescript
   // Only get the fields you need
   const games = await prisma.game.findMany({
     select: {
       id: true,
       winner: true,
       moveCount: true,
     },
   });
   ```

4. **Use middleware for common logic**:
   ```typescript
   prisma.$use(async (params, next) => {
     console.log(`Query: ${params.model}.${params.action}`);
     return next(params);
   });
   ```

## Useful Commands

```bash
npm run db:studio           # Open Prisma Studio GUI
npm run db:migrate          # Create and run migrations
npm run db:push             # Sync schema to database (dev only)
npm run db:reset            # Reset database (careful!)
npx prisma validate         # Check schema validity
npx prisma format           # Format schema file
npx prisma generate         # Regenerate Prisma Client
```

## Troubleshooting

### "Prisma Client does not have a query engine yet"

```bash
npm run db:migrate
# or
npx prisma generate
```

### Schema validation errors

```bash
npx prisma validate
```

### Database out of sync with schema

```bash
npm run db:push
```

### Need a fresh start

```bash
npm run db:reset  # Warning: deletes all data!
```
